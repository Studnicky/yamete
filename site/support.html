Frequently Asked Questions

No impacts are detected

Make sure you have an Apple Silicon Mac (M1/M2/M3/M4). Check that Reactivity is not set too low. Try lowering the Spike Threshold in Accelerometer Tuning. If using microphone detection, grant microphone permission in System Settings > Privacy & Security.

The app detects too many false positives

Increase the Spike Threshold, Crest Factor, and Rise Rate in Accelerometer Tuning. Set Confirmations to 3 or higher. Increase Sensor Consensus to require multiple sensors to agree. Add a Cooldown to space out reactions.

No sound plays on impact

Check the Volume range slider — both thumbs should be above zero. Verify an audio device is selected in the Devices panel. Make sure sounds are not muted at the system level.

Screen flash doesn't appear

Check the Flash Opacity range slider — both thumbs should be above zero. Verify displays are selected in the Devices panel. The flash is gated to the sound clip duration, so audio must also be working.

The app uses too much CPU

Try increasing the Report Interval (e.g., 20ms instead of 10ms) to reduce accelerometer polling rate. Disable sensors you don't need. The app should use less than 2% CPU during idle monitoring.

How do I uninstall?

Drag Yamete.app from Applications to the Trash. To remove preferences: delete ~/Library/Preferences/com.studnicky.yamete.plist. To remove logs: delete ~/Library/Application Support/Yamete/.

Does the app work on Intel Macs?

Partially. Intel Macs lack the BMI286 accelerometer, but microphone-based impact detection works on any Mac with a microphone. Headphone motion also works with compatible AirPods/Beats.

System Requirements

macOS 14.0+ (Sonoma), Apple Silicon recommended. Requires USB entitlement for accelerometer access and optional microphone permission.
