Yamete

Your MacBook reacts when you smack it.

Yes. That is the entire app. Welcome.

macOS 14+ · Apple Silicon · v1.0.0

Yamete sits in your menu bar and watches the built-in accelerometer, the microphone, and — if you're wearing them — your AirPods. When any of them feel a smack, it plays a small sound, flashes a face, and posts an optional reaction. The face has range.

We built this because we wanted to. It runs entirely offline, stores nothing, and will probably not change your life in any measurable way. That's fine.

Features (we took this seriously, somehow)

Three Sensors, One Opinion

Built-in accelerometer, microphone transient detection, and AirPods IMU. A consensus engine fuses them before firing so a passing truck doesn't set the whole thing off.

Sound Clips With Taste

Impact intensity picks the clip — a tap gets a small "mm", a full-force slap gets… more. The volume and clip pool are tunable.

The Face

A live face in the menu bar that reacts on every detected impact, independent of Flash Mode. It has eleven expressions and picks one that matches the hit.

Flash Mode

Three modes: off, full-screen overlay, or notification banner. Pick the one that won't get you fired.

40 Languages

Notification copy ships in 40 locales. You can pick the notification language independently of your system language, for when you want to feel multicultural about it.

Knobs For Days

Sensitivity range, volume range, flash opacity range, per-sensor gate tuning (spike, crest, rise rate, confirmations), cooldown debounce. Tweak until it reads you.

Why though?

Honestly? We wanted to. Apple Silicon MacBooks ship with a real accelerometer — the same BMI286 that's in phones — and Apple provides zero public API to read it. The whole CMMotionManager surface is marked API_UNAVAILABLE(macos). This felt like a dare.

The result is a fully functional reason to touch your laptop too enthusiastically. It's the useless-tool equivalent of building a ship in a bottle: most of the interesting work is invisible, the end result is not especially practical, and the point was the building.

If you use this daily, we'd love to hear why. If you install it, laugh once, and delete it — that's also a completely valid outcome and we are glad you stopped by.

How it works

Each sensor runs its own detection pipeline: raw samples pass through bandpass filtering and a six-gate analysis (spike threshold, rise rate, crest factor, confirmation count, warmup, cooldown). When enough sensors agree within a time window, Yamete plays a sound clip and triggers the visual response, both scaled to impact intensity.

It's a menu bar app (LSUIElement) — no Dock icon, no app windows. All controls live in the menu bar dropdown, because we are not monsters.

Privacy (the boring-but-actually-nice kind)

Zero network connections. All sensor data processed locally, in real time. Microphone audio is analyzed for transient peaks and immediately discarded — nothing is recorded. Logs auto-delete after 24 hours. No accounts, no tracking, no ads, no telemetry, no analytics, no "just this once" exceptions.

If this seems too good to be true, the source code is right there and you're welcome to check.

Requirements

  • macOS 14.0+ (Sonoma)
  • Apple Silicon MacBook (M1/M2/M3/M4) if you want the tactile accelerometer channel
  • Any Mac with a microphone for the sound-based detection path (works on every Mac)
  • AirPods Pro / Beats for headphone motion (optional, but it's fun)

If you're on an Intel Mac, a Mac Studio, a Mac Mini, or an iMac: microphone detection still works great. You just won't get the "physically smacked the laptop" channel, on account of not having a laptop to smack.

Two builds

Yamete ships in two registers:

  • Yamete — the Mac App Store build. Rated 12+, ships with the tame notification copy ("Mm, again?", "Show off~", "OUCH"). Safe for most workplaces, probably.
  • Yamete Direct — a notarized direct download. Ships with spicier notification copy. Not submitted to the App Store. For consenting adults who specifically wanted the spicier one.

Both builds use the same detection engine, settings, sounds, and face library. They differ only in the notification text register.

Accelerometer setup on App Store (optional)

The Mac App Store build runs under App Sandbox. The sandbox silently blocks the IORegistry writes needed to start the built-in accelerometer, which means — out of the box — the App Store build uses the microphone and headphone motion only. Which is fine. The microphone path works great.

If you specifically want the tactile channel on an Apple Silicon MacBook, there's a small open-source helper at docs/sensor-kickstart/ that you can compile with swiftc and install as a LaunchDaemon. The daemon kickstarts the sensor at boot and keeps it live across sleep / wake cycles by re-running the kickstart on every wake event. Yamete then picks up the live stream. Full instructions, safety notes, and diagnostics in the README.

If you install it and it works — or doesn't — please file an issue. We're building a known-working Mac matrix and every data point helps.
