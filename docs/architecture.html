Architecture

How Yamete actually does what it does. For people who would rather read a diagram than trust a privacy policy.

Yamete is more engineered than the concept deserves. Three parallel sensor pipelines, a six-gate per-adapter detector, a consensus fusion engine, and three response modalities. All of it Swift strict concurrency, all of it offline, all of it firing in under 50ms from impact to sound.

What follows is the real code path, annotated for a technical audience. The diagrams are generated from the actual class and method structure, not a marketing approximation of it.

  • IOHIDPublic -- C header shim exposing private IOKit HID types Apple doesn't publish. Read-only; never changes.
  • SensorKit -- the entire sensor abstraction: HID reader, mic tap, AirPods motion, signal filters, per-sample detector, fan-in task group.
  • YameteCore -- shared types and settings. No sensor knowledge, no UI. Imported by everything.
  • ResponseKit -- audio playback, screen overlay, notification dispatch. Knows about impact intensity; knows nothing about where that intensity came from.
  • YameteApp -- the orchestrator. Connects pipeline to response. Also the SwiftUI menu bar shell.

Full signal pipeline (sensor to speaker)

Three sensor streams run concurrently in a Swift task group. Each produces SensorImpact events independently. A fan-in layer collects them. The fusion engine gates consensus and rearm timing. ImpactController maps the fused intensity through a user sensitivity window and dispatches response.

Per-sensor detection gates (ImpactDetector, all six)

Every adapter runs an ImpactDetector independently. Each sample passes through six gates in order. All six must pass to produce an intensity value. Any gate failure returns nil for that sample -- no event emitted, no noise downstream.

The combination of these gates is what makes Yamete usable at a desk. Typing produces magnitude but no sharp rise. Footsteps produce a rise but the crest factor is too low (ambient RMS is already elevated). A real smack produces all six: a clean spike, a fast transient, a high crest, and multiple samples above threshold in the window.

Fusion engine (ImpactFusionEngine, consensus + rearm)

Individual sensor impacts feed into a fusion engine that enforces two additional constraints before allowing a response to fire: consensus (enough sensors must agree within a time window) and rearm (minimum gap between consecutive responses).

The default consensusRequired = 1 means any single sensor can trigger a response. Set it to 2 and both the accelerometer and microphone have to agree within 150ms before anything fires -- eliminates most false positives at the cost of some missed detections on very clean impacts.

Sensitivity gate (ImpactController, pre-response)

After fusion, intensity passes through one more mapping before dispatch. The user's sensitivity sliders define a window over the [0,1] intensity range. The gate rejects impacts below the lower edge and linearly rescales survivors to fill [0,1] for the response layer. This is what makes "sensitivity" feel like a threshold rather than a volume knob.

Response dispatch (three paths, all MainActor)

Concurrency model (threads, locks, actors)

Three different thread contexts produce sensor data. A single lock per adapter serializes state. The fan-in task group bridges to Swift concurrency. The main actor serializes all response work.

The patch in 1.1.1 fixed a recursive lock crash here: continuation.finish() and continuation.yield() were being called inside state.withLock {}. AsyncThrowingStream._Storage acquires its own os_unfair_lock inside those methods. Non-reentrant lock, double-acquisition, abort cause 89859. The fix captures the continuation reference as the lock's return value, releases the lock, then calls the method.

Pipeline lifecycle (ImpactController state machine)

ImpactController manages a start/stop pipeline keyed to whether any response output is enabled. If both sound and visual are off, the pipeline stops entirely. Settings changes rebuild the pipeline from scratch with fresh adapters.
